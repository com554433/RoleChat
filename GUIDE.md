# RoleChat 全面介绍

基于 React + Electron 的桌面角色扮演应用，支持深度角色创建、声音定制、多角色管理。

## 核心功能

| 功能 | 说明 |
|------|------|
| 一键创建角色 | 输入角色名 + 作品名，自动生成深度角色设定、开场白、语音风格 |
| 扮演角色语音合成 | 上传声音样本，程序用该音色朗读回复；支持自动朗读和手动播放暂停 |
| 多角色管理 | 左侧微信风格列表，每个角色独立聊天记录，一键切换 |
| 流式回复 | 实时打字效果，逐字显示 |
| 思考过程 | 可折叠查看推理过程 |
| 深色/浅色 | 一键切换主题 |
| 桌面 EXE | 单文件 Electron 应用，不受浏览器跨域限制 |

## 设置详解

点击右上角齿轮图标打开设置面板，包含以下区块：

### 账号设置

| 字段 | 默认值 | 说明 |
|------|--------|------|
| 服务密钥 | (需填写) | 服务商凭证，支持粘贴/复制按钮 |
| 服务地址 | (需填写) | 完整访问地址 |
| 对话版本 | `mimo-v2.5-pro` | 对话服务版本 |
| 语音版本 | `mimo-v2.5-tts-voiceclone` | 语音合成版本 |

### 备用账号

独立开关，与上方主账号完全隔离：

| 字段 | 说明 |
|------|------|
| 开关 | 开启后对话请求走备用账号 |
| 语音沿用主账号 | 默认开启，语音合成走上方主账号配置 |

### 角色设置

- 编辑角色名称和设定文本
- 上传角色头像（支持图片文件和粘贴）
- 上传聊天背景
- 上传语音样本（wav/mp3）

### 其他

- 思考过程开关
- 语音朗读开关
- 深色/浅色主题切换
- 清空当前聊天记录

## 角色创建方法

系统采用角色行为建模方法。用户只需输入角色名和作品名，系统自动生成包含以下要素的深度角色配置：

| 要素 | 说明 |
|------|------|
| 行为偏好 | 角色注意力倾向（先注意什么、忽略什么） |
| 反应规则 | 不同情境下的行为模式（靠近/沉默/回避） |
| 表达风格 | 句长、自称、口头禅、情绪流露方式 |
| 关系模式 | 对亲近者与陌生人的差异 |
| 决策原则 | 价值冲突时的优先级和底线 |
| 诚实边界 | 不确定时直接承认，不编造 |

### 输出格式

生成的角色设定采用结构化文本格式：

```yaml
---
name: 角色中文名
voice_style: 语音风格描述（10字以内）
greeting: 开场白（1-2句，角色语气）
---
# 角色名 · 一句话核心标签

> "角色代表性台词"

## 身份
[第一人称自我介绍，50字]

## 行为动态
- 日常：[默认反应]
- 压力下：[情绪波动时的变化]

## 表达质感
- 句式：[长短句偏好]
- 口头禅：[2-3个高频词]
- 自称：[我/人家/本小姐等]

## 核心矛盾
[最重要的内在矛盾]

## 对话样本
- 场景A：用户说"..." → 回应"..."
- 场景B：用户说"..." → 回应"..."
```

## 声音定制

基于语音合成技术，实现音色复现 + 语音朗读：

### 使用流程

1. 准备一段角色的语音样本（建议 10-30 秒，清晰的单人声音）
2. 在设置面板上传该音频文件
3. 开启语音开关
4. 发送消息后，回复自动朗读；也可点击消息气泡上的播放按钮手动播放

### 技术说明

- 语音样本以 base64 编码存储在本地
- 返回音频数据直接转为 Audio 对象播放
- 超过 500KB 的语音样本不会持久化（避免存储溢出）

### 自动朗读 vs 手动播放

- **自动朗读**：流式回复完成后，自动请求语音合成并播放
- **手动播放**：点击消息气泡下方的朗读按钮
- 缓存机制：同一消息只合成一次语音，后续点击直接播放缓存

## 技术架构

### 技术栈

```
React 18          UI 框架
TypeScript        类型安全
Zustand           状态管理（含 persist 中间件）
Tailwind CSS      样式
Vite 5            构建工具
Electron 28       桌面壳
MiMo V2.5         后端服务
```

### 数据流

```
用户输入 → ChatInput
  → callLLM (services/api.ts)
    → fetch (SSE 流式)
      → readSSEStream
        → updateLastMessage (chatStore)
          → MessageBubble 实时渲染
            → (可选) callVoiceClone → Audio 播放
```

### 状态管理

Zustand store (`chatStore.ts`) 持久化字段：

| 字段 | 说明 |
|------|------|
| `skills` | 所有角色（含配置、头像、语音样本） |
| `skillChats` | 每个角色的聊天记录 key-value |
| `apiSettings` | 主账号配置 |
| `nonTokenPlan` | 备用账号配置 |
| `theme` / `showThinking` / `isTtsEnabled` | UI 偏好 |

不持久化字段：`isLoading`、`isSettingsOpen`、`isSkillImporterOpen`

### 性能优化

- 所有组件使用 Zustand selector 逐个订阅
- 全部 7 个组件包裹 `React.memo`
- Vite 代码分割：react / zustand 独立 chunk
- 聊天记录截断至最近 200 条
- 头像 > 300KB 和语音样本 > 500KB 不持久化
- ChatInput 用 `requestAnimationFrame` 避免布局抖动

## 开发与构建

```bash
# 安装依赖
npm install

# 浏览器开发（端口 5173）
npm run dev

# Electron 开发
npm run electron:dev

# TypeScript 编译检查
npx tsc -b

# 构建 + 打包 EXE + 部署到 G 盘
.\build-exe.bat

# 仅打包 EXE
npm run electron:build
```

构建输出：
- `dist/` — Vite 构建产物
- `release/win-unpacked/` — Electron 解包目录
- `release/RoleChat.exe` — 单文件 EXE（约 70MB）
